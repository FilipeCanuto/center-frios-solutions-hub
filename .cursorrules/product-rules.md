# 🚨 REGRA GLOBAL OPERACIONAL: CRIAÇÃO E EDIÇÃO DE PRODUTOS 🚨

ATENÇÃO AGENTE: A partir de agora, toda vez que você for solicitado a criar, editar ou atualizar uma página ou seção referente a qualquer produto deste projeto, você DEVE, OBRIGATORIAMENTE, seguir o fluxo de trabalho exato abaixo antes de escrever qualquer código ou inserir qualquer asset visual.

## PASSO 1: LOCALIZAÇÃO DO DIRETÓRIO DO PRODUTO

Identifique qual é o produto alvo da tarefa e busque no repositório a pasta específica dedicada a ele (ex: `.imagens_[NomeDoProduto]`, `assets/[NomeDoProduto]`, etc.).

## PASSO 2: LEITURA DE DOSSIÊ TÉCNICO E COPY (.md)

Antes de gerar qualquer texto, procure na pasta do produto por arquivos `.md` que contenham "dossier", "info" ou "copy" no nome. Leia este arquivo para absorver os argumentos de venda, especificações técnicas e o tom de voz correto (Hard Copy, Foco em Conversão). Não invente especificações; use estritamente os dados do dossiê.

## PASSO 3: O MAPA DE IMAGENS (LEITURA OBRIGATÓRIA PARA ASSETS)

NUNCA escolha uma imagem apenas adivinhando pelo nome do arquivo.

> Dentro da pasta de imagens do produto, procure por um arquivo `.md` de mapeamento (ex: `mapa-de-imagens.md`, `index-imagens.md`).

Você deve ler este arquivo para entender qual imagem visual corresponde a qual bloco da página.

_Exemplo:_ Se você vai criar a seção sobre "Painel de Controle", consulte o mapa `.md` para encontrar o nome exato do arquivo de imagem que mostra o painel, e utilize exclusivamente essa imagem no código.

## PASSO 4: EXECUÇÃO

Apenas após completar os Passos 1, 2 e 3, inicie a escrita do código (HTML/React/Tailwind/etc.). Garanta que as imagens inseridas correspondam perfeitamente à semântica definida no mapa de imagens e que o texto respeite o dossiê.
